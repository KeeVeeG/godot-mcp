# Prerequisites for Resource Configuration Test Plan

> **Source test plan:** `resource_config_test_plan.md`  
> **Source tool:** `server/src/tools/resource_config.ts`  
> **Godot handler:** `addons/godot_mcp/commands/resource_config_commands.gd`  
> **Tools covered:** `get_resource_types`, `get_resource_properties`, `create_resource_from_template`, `import_resource`, `get_resource_import_settings`, `set_resource_import_settings`  
> **Generated:** 2026-07-08

---

## Required Project State

- **Godot 4.x project** exists on disk with a valid `project.godot` file. The Godot editor must be open (any project type works â€” 2D, 3D, or empty). The tools treat resource operations at the project level, not scene-dependent.
- **godot_mcp addon is installed and active.** The addon lives at `res://addons/godot_mcp/` and must be enabled in **Project â†’ Project Settings â†’ Plugins**. The plugin registers the autoload `mcp_runtime` at `res://addons/godot_mcp/services/mcp_runtime.gd` in `project.godot`'s `[autoload]` section.
- **MCP server is running** (`node server/dist/index.js` or `npx -y @keeveeg/godot-mcp`) and connected to the Godot editor via WebSocket on one of ports 6505-6514. The Godot editor's **MCP** bottom panel shows "Connected" status.
- **Dedicated test project strongly recommended.** The `create_resource_from_template` tool writes `.tres` files to disk. The `import_resource` and `set_resource_import_settings` tools modify `.import` files and trigger reimports. Use a project where these side effects are acceptable. Do NOT run in a production project.
- **`res://test_output/` directory must exist** (or be auto-creatable by Godot). The `create_resource_from_template` handler calls `MCPCommandHelpers.ensure_dir()` internally, but having it pre-created verifies permissions. Tests from scenarios S1, S3, S5, S17 and Integration 3 write to this directory.

---

## Required Scenes

**No specific scene hierarchy is required.** All resource configuration tools operate on the engine's resource system (`ClassDB`, `ResourceLoader`, `ResourceSaver`, `EditorFileSystem`), which is independent of the scene tree. The test executor can have any scene open (including no scene at all).

If running resource_config tests alongside tests from other test plans that require specific scenes, any scene will do â€” these tools are scene-agnostic. However, a minimal empty scene is recommended for cross-plan compatibility:

- **File path:** `res://test_scenes/empty.tscn` (optional convenience)
- **Root node:** `Node2D` named `"Root"`
- **Hierarchy:** Empty root scene â€” no child nodes required.

## Required Resources on Disk

The following files must exist in the project's `res://` namespace BEFORE running their dependent scenarios. The GDScript handler for `_import`, `_get_import_settings`, and `_set_import_settings` all call `FileAccess.file_exists(path)`, and the import settings tools look for a sibling `.import` file. Files that need `.import` files must have been imported by Godot at least once.

### 1. `res://assets/textures/icon.svg`
- **Type:** SVG image (default Godot icon)
- **Required for:** `import_resource` S1, S2, S3, S6, S12, S13, S14 | `get_resource_import_settings` S1, S10 | `set_resource_import_settings` S1, S2, S5, S6, S10, S13, S14, S15, S16, S17, S18 | Integration 2, Integration 4
- **Must have:** A `.import` file (`res://assets/textures/icon.svg.import`) generated by Godot's import pipeline. Without it, `get_resource_import_settings` returns `"No .import file found"` and `set_resource_import_settings` fails.
- **How to create:** Create a new Godot project â€” `icon.svg` is included by default at `res://icon.svg`. Copy it to `res://assets/textures/icon.svg`:
  ```gdscript
  # Via godot_execute_editor_script:
  DirAccess.make_dir_recursive_absolute("res://assets/textures")
  DirAccess.copy_absolute("res://icon.svg", "res://assets/textures/icon.svg")
  ```
  Then trigger a reimport:
  ```
  godot_import_resource({ "path": "res://assets/textures/icon.svg" })
  ```

### 2. `res://assets/sounds/explosion.ogg`
- **Type:** OGG Vorbis audio file
- **Required for:** `import_resource` Scenario 4
- **Must have:** Any valid `.ogg` file at this path. Does NOT need a `.import` file for `import_resource` (it just triggers reimport), but having one is fine.
- **How to create:** Place any `.ogg` audio file at `res://assets/sounds/explosion.ogg`. A minimal valid OGG can be generated or downloaded. If you do not have one, you can:
  - Download a free sound effect from freesound.org, OpenGameArt.org, or similar
  - Use a silent/tiny `.ogg` file for testing purposes
  - Create it via Godot's audio bus recording (advanced)

### 3. `res://assets/sounds/music.ogg`
- **Type:** OGG Vorbis audio file
- **Required for:** `get_resource_import_settings` Scenario 2 | `set_resource_import_settings` Scenario 3
- **Must have:** A `.import` file (`res://assets/sounds/music.ogg.import`) for the import settings tools to read/modify. Without it, `get_resource_import_settings` returns `"No .import file found"`.
- **How to create:** Same as explosion.ogg above. Once placed, trigger import:
  ```
  godot_import_resource({ "path": "res://assets/sounds/music.ogg" })
  ```
  This ensures Godot generates the `.import` file.

### 4. `res://assets/models/character.fbx`
- **Type:** FBX 3D model (or `.glb` / `.gltf`)
- **Required for:** `import_resource` Scenario 5 | `get_resource_import_settings` Scenario 3 | `set_resource_import_settings` Scenario 4
- **Must have:** A `.import` file (`res://assets/models/character.fbx.import`) for import settings tools. A simple cube/skeleton model suffices.
- **How to create:** Export a minimal character model from Blender or any 3D tool as `.fbx`. Place at `res://assets/models/character.fbx`. Trigger import:
  ```
  godot_import_resource({ "path": "res://assets/models/character.fbx" })
  ```

### 5. `res://assets/existing_material.tres`
- **Type:** `.tres` resource file â€” `StandardMaterial3D` (or compatible material)
- **Required for:** `create_resource_from_template` Scenario 2 (happy path with template)
- **Must have:** A valid `.tres` file containing a `StandardMaterial3D` resource with some non-default properties (so the template is distinguishable from defaults).
- **How to create:**
  ```
  godot_create_resource({ "type": "StandardMaterial3D", "path": "res://assets/existing_material.tres", "properties": { "albedo_color": "#FF4444FF", "metallic": 0.8, "roughness": 0.3 } })
  ```
  Or via editor script:
  ```gdscript
  var mat = StandardMaterial3D.new()
  mat.albedo_color = Color(1.0, 0.27, 0.27, 1.0)  # red
  mat.metallic = 0.8
  mat.roughness = 0.3
  DirAccess.make_dir_recursive_absolute("res://assets")
  ResourceSaver.save(mat, "res://assets/existing_material.tres")
  ```

### 6. `res://materials/some_shader_material.tres`
- **Type:** `.tres` resource file â€” `ShaderMaterial`
- **Required for:** `create_resource_from_template` Scenario 4 (template-based ShaderMaterial creation)
- **Must have:** A valid `ShaderMaterial` resource referencing an existing shader. The shader itself must also exist.
- **How to create:**
  1. First create a shader at `res://shaders/test_shader.gdshader`:
     ```
     godot_create_shader({ "path": "res://shaders/test_shader.gdshader", "type": "canvas_item" })
     ```
  2. Then create the ShaderMaterial referencing it:
     ```gdscript
     # Via godot_execute_editor_script:
     var shader = load("res://shaders/test_shader.gdshader")
     var mat = ShaderMaterial.new()
     mat.shader = shader
     DirAccess.make_dir_recursive_absolute("res://materials")
     ResourceSaver.save(mat, "res://materials/some_shader_material.tres")
     ```

### 7. `res://scripts/some_script.gd`
- **Type:** GDScript file (NOT a resource â€” used as an invalid template)
- **Required for:** `create_resource_from_template` Scenario 12 (template is a script, not a resource)
- **Must have:** Any valid `.gd` file. The point is that it's not a `.tres` resource template.
- **How to create:**
  ```
  godot_create_script({ "path": "res://scripts/some_script.gd", "content": "extends Node\n\nfunc _ready() -> void:\n\tpass\n" })
  ```

### 8. `res://assets/some_resource.tres`
- **Type:** `.tres` resource file â€” any valid resource type (e.g., `StyleBoxFlat`, `Theme`, or `StandardMaterial3D`)
- **Required for:** `get_resource_import_settings` Scenario 4 (tests querying import settings on a native `.tres` resource, which may not have a `.import` file)
- **Must have:** The `.tres` file must exist. It may or may NOT have a `.import` file â€” the scenario specifically tests behavior on non-imported (native) resources.
- **How to create:**
  ```
  godot_create_resource({ "type": "StyleBoxFlat", "path": "res://assets/some_resource.tres" })
  ```

- **No blocking dialogs** are open in the Godot editor. The plugin auto-dismisses most dialogs during WebSocket operations, but manual dialogs (e.g., "Save changes?") can block responses.
- **Godot Output log and MCP panel visible** for debugging. Check the **MCP** bottom panel for `[MCP]` connection and tool execution messages.
- **No breakpoints set** on any GDScript files. Breakpoints are not relevant to resource configuration tools, which do not execute user scripts.
- **No specific editor layout, tool selection, or viewport configuration required.** The resource type system is independent of the editor's visual state.

## Required Settings/Config

- **No specific `project.godot` settings** are required beyond what the godot_mcp plugin registers automatically. Resource types, import settings, and file operations do not need any custom project settings.
- **No specific input actions, collision layers, or custom autoloads** are needed. The `mcp_runtime` autoload registered by the plugin is automatically present. No other autoloads are required.
- **`godot_mcp_config.json`** (optional, at project root): If present, ensure none of these resource config tools are disabled:
  ```json
  {
    "enabled_tools": {
      "get_resource_types": true,
      "get_resource_properties": true,
      "create_resource_from_template": true,
      "import_resource": true,
      "get_resource_import_settings": true,
      "set_resource_import_settings": true
    }
  }
  ```
  If the file does not exist, all tools are enabled by default.

- **Cross-tool dependencies for setup:** The following tools from OTHER modules are needed to create prerequisite files (used ONLY for setup, not part of the test plan itself):
  - `godot_create_resource` (from `resource` module) â€” to create `.tres` template files
  - `godot_create_script` (from `script` module) â€” to create the `.gd` file for S12
  - `godot_create_shader` (from `shader` module) â€” to create the shader for ShaderMaterial template
  - `godot_import_resource` (from `resource_config` itself) â€” to trigger initial imports and generate `.import` files

  Ensure these tools are NOT disabled in `godot_mcp_config.json`:
  ```json
  {
    "enabled_tools": {
      "create_resource": true,
      "create_script": true,
      "create_shader": true
    }
  }
  ```

## Setup Script

This GDScript (executed via `godot_execute_editor_script`) creates ALL prerequisite directories and files in one go. Run it once at the start of a test session:

```gdscript
# ============================================================
# Resource Config Test Plan â€” Full Prerequisite Setup
# Run via: godot_execute_editor_script
# ============================================================

# --- Create directories ---
DirAccess.make_dir_recursive_absolute("res://assets/textures")
DirAccess.make_dir_recursive_absolute("res://assets/sounds")
DirAccess.make_dir_recursive_absolute("res://assets/models")
DirAccess.make_dir_recursive_absolute("res://materials")
DirAccess.make_dir_recursive_absolute("res://scripts")
DirAccess.make_dir_recursive_absolute("res://shaders")
DirAccess.make_dir_recursive_absolute("res://test_output")

# --- 1. Copy default icon.svg to assets/textures ---
if not FileAccess.file_exists("res://assets/textures/icon.svg") and FileAccess.file_exists("res://icon.svg"):
	DirAccess.copy_absolute("res://icon.svg", "res://assets/textures/icon.svg")
	print("[SETUP] Copied icon.svg to res://assets/textures/")

# --- 2. Create explosion.ogg placeholder (minimal valid OGG) ---
# A truly minimal OGG file. If you have a real .ogg, copy it instead.
if not FileAccess.file_exists("res://assets/sounds/explosion.ogg"):
	var f = FileAccess.open("res://assets/sounds/explosion.ogg", FileAccess.WRITE)
	if f:
		# Minimal OGG container (OggS magic + empty page)
		var ogg_header = PackedByteArray([
			0x4F, 0x67, 0x67, 0x53, 0x00, 0x02, 0x00, 0x00,
			0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00,
			0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00,
			0x00, 0x00, 0x01, 0x1E, 0x01, 0x76, 0x6F, 0x72,
			0x62, 0x69, 0x73, 0x00, 0x00, 0x00, 0x00, 0x02,
			0x44, 0xAC, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00,
			0x00, 0xF0, 0x01, 0x00, 0x00, 0x00, 0x00, 0x00,
			0xB8, 0x01
		])
		f.store_buffer(ogg_header)
		f.close()
		print("[SETUP] Created minimal explosion.ogg")
	else:
		print("[SETUP] WARNING: Could not create explosion.ogg â€” create manually")

# --- 3. Create music.ogg (copy explosion.ogg or create separate) ---
if not FileAccess.file_exists("res://assets/sounds/music.ogg"):
	if FileAccess.file_exists("res://assets/sounds/explosion.ogg"):
		DirAccess.copy_absolute("res://assets/sounds/explosion.ogg", "res://assets/sounds/music.ogg")
		print("[SETUP] Copied explosion.ogg â†’ music.ogg")
	else:
		print("[SETUP] WARNING: Could not create music.ogg â€” create manually")

# --- 4. Create a minimal character.fbx (or use a GLB) ---
if not FileAccess.file_exists("res://assets/models/character.fbx"):
	# FBX is complex to generate programmatically. Create a minimal GLB instead,
	# then rename â€” Godot handles GLB natively. Or skip if unavailable.
	# Alternative: create a simple .obj file
	var obj_content = "# Minimal OBJ for testing\nv 0 0 0\nv 1 0 0\nv 1 1 0\nv 0 1 0\nf 1 2 3\nf 1 3 4\n"
	var f = FileAccess.open("res://assets/models/character.fbx", FileAccess.WRITE)
	if f:
		f.store_string(obj_content)
		f.close()
		print("[SETUP] Created placeholder character.fbx (OBJ content)")
	else:
		print("[SETUP] WARNING: Could not create character.fbx â€” create manually")

# --- 5. Create existing_material.tres (StandardMaterial3D template) ---
if not FileAccess.file_exists("res://assets/existing_material.tres"):
	var mat = StandardMaterial3D.new()
	mat.albedo_color = Color(1.0, 0.27, 0.27, 1.0)  # Red
	mat.metallic = 0.8
	mat.roughness = 0.3
	mat.emission_enabled = true
	mat.emission = Color(0.0, 0.0, 0.0, 1.0)
	mat.emission_energy_multiplier = 1.0
	var err = ResourceSaver.save(mat, "res://assets/existing_material.tres")
	if err == OK:
		print("[SETUP] Created existing_material.tres (red StandardMaterial3D)")
	else:
		print("[SETUP] WARNING: Failed to save existing_material.tres: ", error_string(err))

# --- 6. Create test shader + ShaderMaterial template ---
if not FileAccess.file_exists("res://shaders/test_shader.gdshader"):
	var shader_code = """shader_type canvas_item;

uniform vec4 modulate_color : source_color = vec4(1.0, 1.0, 1.0, 1.0);
uniform float brightness : hint_range(0.0, 2.0) = 1.0;

void fragment() {
	vec4 tex = texture(TEXTURE, UV);
	COLOR = tex * modulate_color * brightness;
}
"""
	var f = FileAccess.open("res://shaders/test_shader.gdshader", FileAccess.WRITE)
	if f:
		f.store_string(shader_code)
		f.close()
		print("[SETUP] Created test_shader.gdshader")
	else:
		print("[SETUP] WARNING: Could not create shader")

if not FileAccess.file_exists("res://materials/some_shader_material.tres"):
	if ResourceLoader.exists("res://shaders/test_shader.gdshader"):
		var shader = load("res://shaders/test_shader.gdshader")
		var smat = ShaderMaterial.new()
		smat.shader = shader
		smat.set_shader_parameter("brightness", 1.5)
		var err = ResourceSaver.save(smat, "res://materials/some_shader_material.tres")
		if err == OK:
			print("[SETUP] Created some_shader_material.tres (ShaderMaterial)")
		else:
			print("[SETUP] WARNING: Failed to save shader material: ", error_string(err))
	else:
		print("[SETUP] WARNING: Shader file missing â€” shader material not created")

# --- 7. Create some_script.gd (for invalid template test) ---
if not FileAccess.file_exists("res://scripts/some_script.gd"):
	var script_code = """extends Node

## A simple test script used as an invalid resource template.
func _ready() -> void:
	pass
"""
	var f = FileAccess.open("res://scripts/some_script.gd", FileAccess.WRITE)
	if f:
		f.store_string(script_code)
		f.close()
		print("[SETUP] Created some_script.gd")
	else:
		print("[SETUP] WARNING: Could not create some_script.gd")

# --- 8. Create some_resource.tres (native resource for import settings test) ---
if not FileAccess.file_exists("res://assets/some_resource.tres"):
	var sb = StyleBoxFlat.new()
	sb.bg_color = Color(0.2, 0.2, 0.8, 1.0)  # Blue
	sb.border_width_left = 2
	sb.border_width_right = 2
	sb.border_width_top = 2
	sb.border_width_bottom = 2
	sb.corner_radius_top_left = 8
	sb.corner_radius_top_right = 8
	sb.corner_radius_bottom_left = 8
	sb.corner_radius_bottom_right = 8
	var err = ResourceSaver.save(sb, "res://assets/some_resource.tres")
	if err == OK:
		print("[SETUP] Created some_resource.tres (StyleBoxFlat)")
	else:
		print("[SETUP] WARNING: Failed to save some_resource.tres: ", error_string(err))

# --- 9. Create already_exists.tres for overwrite test (Scenario 17) ---
if not FileAccess.file_exists("res://test_output/already_exists.tres"):
	var mat = StandardMaterial3D.new()
	mat.albedo_color = Color(0.0, 1.0, 0.0, 1.0)  # Green
	mat.metallic = 0.0
	mat.roughness = 1.0
	var err = ResourceSaver.save(mat, "res://test_output/already_exists.tres")
	if err == OK:
		print("[SETUP] Created test_output/already_exists.tres (green StandardMaterial3D)")
	else:
		print("[SETUP] WARNING: Failed to save already_exists.tres: ", error_string(err))

# --- Done ---
print("[SETUP] Resource Config prerequisites complete. Refresh the filesystem.")
```

```

```

```

godot_import_resource({ "path": "res://assets/sounds/music.ogg" })
godot_import_resource({ "path": "res://assets/models/character.fbx" })
```

Verify `.import` files exist:
